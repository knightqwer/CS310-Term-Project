# Firestore Schema

## Collections

### `users/{uid}`

Created when a user registers. The `uid` matches the Firebase Auth UID.

| Field | Type | Description |
|-------|------|-------------|
| `displayName` | `String` | User's full name |
| `email` | `String` | User's email address |
| `bio` | `String` | Short profile bio |
| `photoURL` | `String` | URL of the user's avatar image (empty string when unset) |
| `attendingCount` | `int` | Number of events the user is currently registered for |
| `eventsCreated` | `int` | Total number of events the user has created |

---

### `events/{eventId}`

One document per campus event. `eventId` is auto-generated by Firestore.

| Field | Type | Description |
|-------|------|-------------|
| `title` | `String` | Event title |
| `imageUrl` | `String` | URL of the event cover image |
| `status` | `String` | `"upcoming"` or `"past"` |
| `dateTime` | `Timestamp` | Date and time of the event |
| `location` | `String` | Venue / building name |
| `organizer` | `String` | Display name of the organizer |
| `organizerUid` | `String` | UID of the organizer |
| `attendeeCount` | `int` | Current number of registered attendees |
| `maxAttendees` | `int` | Maximum allowed attendees |
| `description` | `String` | Full event description |
| `category` | `String` | Primary category (Academic, Sports, Social, etc.) |
| `tags` | `List<String>` | Additional topic tags |
| `attendeeUids` | `List<String>` | UIDs of all registered attendees |
| `createdBy` | `String` | UID of the user who created the event |
| `createdAt` | `Timestamp` | When the document was created |

---

### `events/{eventId}/messages/{messageId}`

Subcollection — chat messages for an event. `messageId` is auto-generated.

| Field | Type | Description |
|-------|------|-------------|
| `senderUid` | `String` | UID of the sender |
| `senderName` | `String` | Display name of the sender |
| `text` | `String` | Message body |
| `createdAt` | `Timestamp` | When the message was sent |

---

### `users/{uid}/notifications/{notifId}`

Subcollection — per-user notifications. `notifId` is auto-generated.

| Field | Type | Description |
|-------|------|-------------|
| `title` | `String` | Notification title |
| `body` | `String` | Notification body text |
| `eventId` | `String?` | Related event ID (nullable) |
| `read` | `bool` | Whether the user has read the notification |
| `createdAt` | `Timestamp` | When the notification was created |

---

### `reports/{reportId}`

Top-level collection for user reports. `reportId` is auto-generated.

| Field | Type | Description |
|-------|------|-------------|
| `reporterUid` | `String` | UID of the user who filed the report |
| `reportedUid` | `String` | UID of the reported user |
| `reason` | `String` | Reason text provided by the reporter |
| `createdAt` | `Timestamp` | When the report was submitted |

---

See `firestore.rules` in the project root.

## Query Patterns

| Use case | Query |
|----------|-------|
| My upcoming registered events | `events` where `attendeeUids` array-contains `uid`, filter client-side by `dateTime >= now` |
| My created events | `events` where `createdBy == uid` |
| My past events | `events` where `attendeeUids` array-contains `uid`, filter client-side by `dateTime < now` |
| Event chat messages (ordered) | `events/{eventId}/messages` order by `createdAt asc` |
| User notifications (unread) | `users/{uid}/notifications` where `read == false` |
